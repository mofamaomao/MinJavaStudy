## 核心概念（我的总结）

### 游戏循环的本质
游戏每一帧执行：更新逻辑 → 渲染画面
这个循环由框架驱动，每帧调用一次 update()

### ApplicationListener 生命周期
init() → update() → pause() → resume() → dispose() → exit()
- init()：游戏启动，初始化资源
- update()：每帧调用，游戏循环的执行者
- pause()/resume()：失焦/恢复（切换应用等）
- dispose()：释放内存、关闭文件等资源
- exit()：退出游戏

### 继承链
ApplicationListener（Arc接口）
  └── ApplicationCore（抽象类，Arc）
        └── ClientLauncher（Mindustry具体实现）

### 帧率控制（锁帧）
- 目标帧率如60fps → 每帧间隔约16ms
- 每帧逻辑执行完后，用 sleep() 等待到下一帧时间点
- 作用：让游戏行为与硬件性能无关，保证公平性
- 慢电脑超时时：不执行sleep，尽力跑，帧率自然下降

### 为什么要锁帧
若不锁帧，子弹"每帧移动5px"：
- 快电脑(10000fps)：每秒移动50000px
- 慢电脑(30fps)：每秒移动150px
→ 多人游戏中产生严重不公平，逻辑结果不一致

## 关联源码

### ClientLauncher.java（核心帧率控制逻辑）
路径：`core/src/mindustry/ClientLauncher.java`
\```java
public void update(){
    int targetfps = ios ? 0 : Core.settings.getInt("fpscap", 120);
    boolean limitFps = targetfps > 0 && targetfps <= 240;
    
    if(limitFps && !changed){
        nextFrame += (1000 * 1000000) / targetfps; // 计算下一帧时间点
    }else{
        nextFrame = Time.nanos();
    }
    
    // 游戏主更新
    asyncCore.begin();
    super.update();
    asyncCore.end();
    
    // 锁帧：等待到下一帧时间点
    if(limitFps){
        long current = Time.nanos();
        if(nextFrame > current){
            long toSleep = nextFrame - current;
            Threads.sleep(toSleep / 1000000, (int)(toSleep % 1000000));
        }
        // 若超时（慢电脑），不sleep，立即进入下一帧
    }
}
\```
