# 2026-05-11 学习笔记

## 今日主题
Java OOP — 继承与接口在 Mindustry 建筑系统中的应用

## 学习阶段
阶段1 · 第1天

## 核心概念（我的总结）

### Block 的继承链
Content → MappableContent → UnlockableContent → Block
- Content：定义最基础的资源/对象类型
- MappableContent：可以被映射（有ID可查找）
- UnlockableContent：可以被解锁（研究树相关）
- Block：具体的建筑类型

### 继承 vs 接口
- 继承（extends）= "是什么"，单一，描述本质身份
- 接口（implements）= "能做什么"，可以多个，描述额外能力
- Java 单继承限制 → 多个能力只能用接口叠加

### Mindustry 的实际结构
- Block 下面还有中间层（如 Wall、Turret 等）
- 具体建筑继承对应功能的中间类
- 仍然使用继承链，但配合接口控制复杂度
