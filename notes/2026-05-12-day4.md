游戏循环深入 — 模块注册与每帧执行顺序

## 学习阶段
阶段1 · 第4天

## 核心概念（我的总结）

### 完整游戏循环链路
框架每帧调用 ApplicationCore.update()
→ for(listener : modules) listener.update()
→ Logic.update()     // 游戏逻辑、单位更新
→ Control.update()   // 玩家输入处理
→ Renderer.update()  // 渲染最新画面
→ UI.update()        // 界面渲染（画面上层）
→ NetServer.update() // 服务端网络同步
→ NetClient.update() // 客户端网络同步

### 模块注册机制
- 所有核心模块实现 `ApplicationListener` 接口
- 在 `ClientLauncher` 里按顺序用 `add()` 注册
- `ApplicationCore.update()` 遍历所有模块依次调用
- **注册顺序 = 每帧执行顺序**，顺序有严格意义

### 为什么是这个顺序
- Logic先于Renderer：先更新状态，再渲染，玩家看到最新画面
- Control在Logic之后、Renderer之前：输入影响逻辑，但结果在本帧渲染
- Network放最后：状态全部确定后再同步给其他玩家

### Logic与游戏循环的连接
- `Logic implements ApplicationListener`
- 被注册进 `ApplicationCore.modules` 列表
- 每帧由框架驱动，Logic.update() → Groups.update() → 每个实体update()

### ApplicationCore.update()核心代码
```java
@Override
public void update(){
    for(ApplicationListener listener : modules){
        listener.update();
    }
}
```

### 实现ApplicationListener的模块
- `UI`：游戏界面
- `Logic`：游戏逻辑
- `NetServer`：网络服务端
- `NetClient`：网络客户端
- `Renderer`：渲染器
- `Control`：输入、存档、快捷键
