# 2026-05-11 问答记录

## 完整问答

Q1: 如何组织几十种建筑类之间的关系？
A: 用继承，由共同父类提供共性行为，子类各自实现差异行为。

Q2: 差异行为全部用继承解决会遇到什么麻烦？
A: 类关系复杂，耦合增加，复用困难。

Q3: Block 的继承链有几层，每层职责是什么？
A: 4层：Content → MappableContent → UnlockableContent → Block

Q4: "Unlockable"是"不可锁定"还是"能被解锁"？
A: 能被解锁的（配合研究树机制）

Q5: 多层继承解决了复杂度问题吗？
A: 没有，复杂度仍在，Mindustry 也确实还在用继承链。

Q6: Block implements Senseable 用了什么机制？
A: 接口（interface）

Q7: 继承和接口的本质区别？
A: 继承="是什么"（单一身份），接口="能做什么"（可叠加能力）
   Java 单继承限制使接口成为多能力组合的必要手段。
