# 2026-05-12 问答记录（第4天）

## 完整问答

Q1: Logic是怎么被游戏循环驱动的？
A: Logic implements ApplicationListener，
   在ClientLauncher用add()注册，
   ApplicationCore.update()每帧遍历所有模块调用update()

Q2: 除了Logic，还有哪些模块实现了ApplicationListener？
A: UI, Logic, NetServer, NetClient, Renderer, Control

Q3: Logic和Renderer的执行顺序为什么重要？
A: 必须先Logic.update()更新状态，
   再Renderer.update()渲染，否则玩家看到的是上一帧画面

Q4: 模块注册顺序是什么？
A: Logic → Control → Renderer → UI → NetServer → NetClient

Q5: ApplicationCore.update()怎么驱动所有模块？
A: for(listener : modules) listener.update()
   遍历所有注册的ApplicationListener依次调用

Q6: 完整的一帧执行链路是什么？
A: 框架调用ApplicationCore.update()
   → 遍历modules → Logic/Control/Renderer/UI/Net各自update()
   → Logic内部调用Groups.update()
   → 每个实体的update()
