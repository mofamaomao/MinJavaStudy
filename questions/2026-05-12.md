# 2026-05-12 问答记录

## 完整问答

Q1: 游戏框架最核心的骨架应该长什么样？
A: 需要持续刷新画面、响应输入、更新逻辑，用循环实现。

Q2: 每一帧游戏需要做哪些事？
A: 更新逻辑（角色移动、工厂加工、敌人AI等）→ 渲染到屏幕

Q3: ApplicationListener 定义了哪些生命周期方法？
A: init / update / pause / resume / dispose / exit

Q4: update() 在生命周期中扮演什么角色？
A: 游戏循环每帧的执行入口

Q5: ClientLauncher.update() 里 sleep 的作用？
A: 计算本帧剩余时间，sleep等待，锁定帧率，使两帧间隔固定

Q6: 为什么要锁帧？
A: 不锁帧时快慢电脑每秒执行帧数不同，
   导致游戏逻辑结果不一致（如子弹位移），多人游戏不公平

Q7: 慢电脑帧计算超时怎么处理？
A: 不执行sleep，立即进入下一帧，帧率自然降低
